# Source Review Workflow

## Purpose

LifePlanSystem can learn from books, articles, repositories, prompts, documents, and external systems without copying them blindly.

External sources are source material, not authority.

## Review before adoption

Before adopting an idea, review it for:

- evidence quality;
- hidden assumptions;
- security;
- privacy;
- complexity;
- maintainability;
- accessibility;
- failure modes;
- anti-sycophancy compatibility;
- fit with the user's real constraints and goals.

## Output format

A source review should include:

1. Source name.
2. Source type.
3. Why it matters.
4. Key ideas extracted.
5. Ideas to incorporate.
6. Ideas to reject or modify.
7. Risks or failure points.
8. Recommended system changes.
9. Whether a new template, rule, or architecture note is needed.
10. Promotion recommendation.

## Copyright / content handling

Do not store large copyrighted passages.

Store summaries, extracted ideas, design implications, implementation notes, and short references.

## Public export rule

For public repositories, store only generic source-review protocols and non-private notes.
