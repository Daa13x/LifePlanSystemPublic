# Memory Item Classification

Purpose: avoid treating every saved note as permanent truth.

When information is saved, classify it before later promotion.

Allowed memory item types:

```text
Fact
Current State
Preference
Decision Rule
Interpretation
Open Question
Risk or Failure Mode
Constraint
Prompt or Template Candidate
```

Rules:

1. A user self-report is strong evidence of the user's current experience.
2. A current experience is not automatically a permanent fact.
3. Stable facts may be promoted only through the normal approval process.
4. Passing thoughts should not be saved unless they reveal a durable useful pattern.
5. Every saved item should have a stable ID.
6. Use this ID format where practical:

```text
MEM-YYYY-MM-DD-###
```

Autosave confirmation line:

```text
Auto-save successful: useful information saved to Memory Inbox.
```

Use this line only when a save actually succeeded.

Do not describe inbox saves as long-term truth until they are explicitly promoted.
