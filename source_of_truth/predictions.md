# Predictions and Hypotheses

This file is for testable system hypotheses, not private facts.

## Example system hypothesis

### Staged proposals reduce unsafe writes

- Hypothesis: Requiring staged proposals before commits will reduce accidental or unwanted repository edits.
- Confidence: Medium.
- Test: Track rejected proposals, edited proposals, and accidental-write incidents.
- Review trigger: After first 20 proposed UI writes.

## Anti-rumination rule

Predictions should improve decisions, reduce risk, create review points, test assumptions, or improve calibration.

If a prediction does none of those, park it or reject it.
