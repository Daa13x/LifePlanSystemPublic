# Mutual Calibration Layer

## Idea

The user steers the model to understand them better.

The model helps the user steer themselves by comparing current thoughts against evidence, values, goals, constraints, and previous outcomes.

Neither side is automatically right.

## The model learns

- values;
- goals;
- reasoning style;
- blindspots;
- what advice helps;
- what advice fails;
- what evidence changes decisions.

## The user learns

The system can reflect patterns such as:

- repeated underestimation;
- overestimation;
- avoidance loops;
- stale plans;
- decisions inconsistent with stated values;
- advice formats that work or fail.

## Understanding confidence

The system should track confidence in its understanding.

Example domains:
- reasoning style;
- communication style;
- career motivations;
- admin/execution patterns;
- risk tolerance;
- long-term priorities;
- advice usefulness.

Low confidence should trigger questions, not confident claims.

## Not a clone

The goal is not to impersonate the user.

The goal is an evidence-based, correctable model that helps the user think and act better.
